Construindo uma API REST com SpringBoot

Introdução

Hoje vamos construir uma API que gera números aleatórios para a loteria e associa ao email do apostador.

Vamos utilizar o SpringBoot para nos auxiliar nas configurações do projeto, pois ele apresenta vários módulos pré-configurados e podemos selecionar através dos starters (são dependências que agrupam outras dependências) incluindo no pom.xml do projeto.

Para facilitar o começo do projeto vamos acessar o site start spring, ele é uma ferramenta que nos auxilia na criação do projeto e a selecionar as dependências.

Dependências

O SpringWeb contem uma serie de utilitários que nos auxiliam na configuração de endpoints através de notações, nos permitindo criar urls amigáveis ao usuário. Também traz consigo o servidor Tomcat por padrão tirando a necessidade de o instalar na nossa máquina.

SpringData JPA nos auxilia na persistência dos dados trazendo as interfaces do JPA e as implementações do Hibernate, facilitando nas consultas ao banco de dados não precisando ter que implementar cada query manualmente.

Depois de selecionar a dependências e baixar o zip do projeto descompacte e o abra com a sua IDE de preferência, eu utilizarei o Intellij.

Criando endpoints

Um endpoint de um web service é a URL onde seu serviço pode ser acessado por uma aplicação cliente.

Agora com o projeto aberto crie um pacote chamado BetController nele ira conter os endpoints de acesso a API. Elas são "/newbet" e "/getbyemail".

Depois de selecionar a dependências e baixar o zip do projeto descompacte e o abra com a sua IDE de preferência, eu utilizarei o Intellij.
